# rust-redis 待优化事项

## 性能优化

### P1 - 过期扫描策略：全量 O(n) → 随机采样 ✅ 已完成
文件: src/cmd/mod.rs  函数: start_expiration_task
实现: 每轮用 reservoir sampling 随机抽 20 个带 TTL 的 key（最多遍历 100 个），
      过期比例 > 25% 则继续抽，最多 10 轮，每 tick 最多处理 1000 个 key，
      与 DB 大小解耦。ThreadRng 在 await 前显式 drop 保证 Send 安全。

### P2 - repl_backlog 改用非阻塞同步原语 ✅ 已完成
文件: src/cmd/mod.rs  字段: ServerContext::repl_backlog
实现: 改为 Arc<tokio::sync::Mutex<...>>，所有调用点改为 .lock().await。
      info.rs（同步函数）使用 try_lock() 做非阻塞尝试。
      涉及文件: cmd/mod.rs, cmd/replication.rs, cmd/info.rs, bin/server.rs,
               tests/helper.rs, tests/test_move_swapdb.rs,
               tests/test_psync2.rs, tests/aof_hang.rs

### P3 - AOF flush 批量化（everysec 模式）✅ 已完成
文件: src/aof.rs  函数: Aof::append / start_aof_task
实现: 新增 append_nobuf() 方法，只写 BufWriter 不 flush。
      everysec 模式下 start_aof_task 用 tokio::select! 监听消息 + 1 秒 ticker，
      每秒统一执行 flush() + sync_data()，消除每命令一次 syscall。
      同时修复了旧方案中独立 sync_task 与 BufWriter 的竞争问题。

### P4 - ACL 检查改用无锁读取 ✅ 已完成
文件: src/cmd/mod.rs, src/cmd/acl.rs, src/cmd/hello.rs, src/acl.rs
实现: 引入 arc-swap crate，ServerContext::acl 从 Arc<RwLock<Acl>>
      改为 Arc<ArcSwap<Acl>>。
      读取路径（check_access/auth/hello/acl list等）：.acl.load() 原子指针加载，无锁。
      写入路径（SETUSER/DELUSER/LOAD）：.acl.rcu(|old| { clone-modify-return }) 原子替换。
      Acl 结构体新增 #[derive(Clone)]。
      涉及文件: acl.rs, cmd/mod.rs, cmd/acl.rs, cmd/hello.rs,
               bin/server.rs, tests/helper.rs, tests/acl.rs,
               tests/test_reset.rs, tests/test_move_swapdb.rs, tests/aof_hang.rs

### P5 - resp::read_line 零分配整数解析 ✅ 已完成
文件: src/resp.rs  函数: read_integer_line
实现: 改用 fill_buf()/consume() 在字节层面直接解析整数，
      完全消除 String 分配和 UTF-8 验证开销。
      影响每条命令的数组头解析（*N\r\n 热路径）。

### P6 - ServerContext clone 开销：拆分子结构（进行中）
文件: src/cmd/mod.rs  结构体: ServerContext
目标: 将字段归组为更少的子 Arc，clone 从 ~58 次原子操作降至 ~15 次
影响: 高连接数场景（每秒数千新连接）时有明显收益
